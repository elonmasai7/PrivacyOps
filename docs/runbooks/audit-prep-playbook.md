# Audit-Prep Playbook

## Purpose

Prepare an organization workspace for external auditor review or enterprise customer security review using complete, traceable, and current evidence.

## Audit Readiness Inputs

- Active framework scope (Kenya DPA, GDPR, SOC 2 readiness, ISO 27001 readiness)
- Control ownership assignments
- Evidence status and freshness
- Open findings, incidents, and overdue tasks

## Preparation Workflow

1. **Define scope**
   - Confirm in-scope frameworks and reporting window.
   - Confirm legal/compliance reviewer for final pack.
2. **Control and evidence review**
   - Ensure each in-scope control has mapped evidence.
   - Review evidence expiry dates and approval statuses.
3. **Gap closure sprint**
   - Create/assign remediation tasks for missing evidence or failed controls.
   - Resolve high-severity findings or document mitigation plans.
4. **Policy and governance check**
   - Verify required policies exist with current approved versions.
   - Confirm legal-review flags addressed where needed.
5. **Generate exports**
   - Generate framework readiness reports.
   - Export required formats (`PDF`, `DOCX`, `CSV`, `JSON`) as requested.
6. **Trust and sharing workflow**
   - Publish only approved trust-center documents.
   - For private sharing, use controlled access workflow.
7. **Final QA pass**
   - Validate report payload consistency against database state.
   - Validate audit logs capture report generation and major updates.

## Auditor Package Checklist

- Organization profile and scope definition
- Framework readiness reports
- Control completion and gap summary
- Evidence index with references
- Incident summary (open and closed)
- Vendor risk summary
- Legal disclaimer and reviewer sign-off

## Exit Criteria

- All required reports generated from current real data.
- Missing-evidence list accepted or resolved.
- Reviewer confirms package is ready for external sharing.
